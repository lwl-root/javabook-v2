映射结果集（高级）Table 的列名与 PO 类的属性名的不同，只是一个小 case。&lt;resultMap&gt; 真正的作用和价值并非在此，而是在于『关系映射』。## 1. 一对一映射使用自定义映射的第二个，也是更重要的原因是：关联映射 。两张表的一对一映射在类与类的关系上体现为 一个类“持有”另一个类。双向的一对一映射则表现为 两个类互相“持有”。持有方所对应的表为『从表』，被持有方所对应的表为『主表』。association 元素用于 从表 方。association 元素的常用属性有：| 属性 | 说明 || :------------ | :----------------------------------------------------------- || property 属性 | 表示所映射对象的属性名。 || column 属性 | 表示表的列名，此处必为 外键列 。 || javaType 属性 | 表示属性所对应的类型名称。 || select 属性 | 表示为了获取关联对象，所调用的另一条 MyBatis select 语句的 id 。 |第一种写法：传统写法xml&lt;mapper namespace="demo.dao.EmployeeMapper"> &lt;resultMap id="BaseResultMap" type="demo.bean.po.Employee"> &lt;id ... /> &lt;result ... /> ... &lt;association property="department" javaType="demo.bean.po.Department"> &lt;id column="deptno" property="deptno" jdbcType="INTEGER"/> &lt;result column="dname" property="dname" jdbcType="VARCHAR"/> &lt;result column="loc" property="loc" jdbcType="VARCHAR"/> &lt;/association> &lt;select id="..." resultMap="BaseResultMap">&lt;/mapper>第二种写法：调用法xml&lt;mapper namespace="demo.dao.EmployeeMapper"> &lt;resultMap id="BaseResultMap" type="demo.bean.po.Employee"> &lt;id ... /> &lt;result ... /> ... &lt;association property="department" column="deptno" javaType="demo.bean.po.Department" select="demo.dao.DepartmentMapper.selectByPrimaryKey"/> &lt;/resultMap> &lt;select id="..." resultMap="BaseResultMap">&lt;/mapper>## 2. 一对多映射一对多的映射在类与类的关系上体现为：一个类持有另一个类的一个对象，另一个类持有这个类的对象集合。如果一对多映射是单向的，则只会使用到 collection 元素；如果一对多映射是双向的，那么还会使用到前面所说的 association 元素。collection 元素的常用属性有：| 属性 | 说明 || :------------ | :----------------------------------------------------------- || property 属性 | 表示所映射对象的属性名。 || column 属性 | 表示表的列名，此处必为 外键列 || javaType 属性 | 表示属性所对应的类型名称，此处必为某种集合类型 || ofType 属性 | 表示集合当中的单元类型 || select 属性 | 表示为了获取关联对象，所调用的另一条 MyBatis select 语句的 id |第一种写法：xml&lt;resultMap id="xxx" type="Department"> &lt;id column="deptno" property="deptno"/> &lt;result column="dname" property="dname"/> &lt;result column="loc" property="loc"/> &lt;collection property="employeeList" ofType="Employee"> &lt;id column="empno" property="empno"/> &lt;result column="ename" property="ename"/> &lt;result column="job" property="job"/> &lt;result column="mgr" property="mgr"/> &lt;result column="hiredate" property="hiredate"/> &lt;result column="sal" property="sal"/> &lt;result column="comm" property="comm"/> &lt;result column="deptno" property="deptno"/> &lt;/collection>&lt;/resultMap> &lt;select id="selectByPK" parameterType="int" resultMap="xxx"> select dept.deptno deptno, dept.dname dname, dept.loc loc, emp.empno empno, emp.ename ename, emp.job job, emp.mgr mgr, emp.hiredate hiredate, emp.sal sal, emp.comm com from dept, emp where dept.deptno = emp.deptno and dept.deptno = #{deptno} &lt;/select>第二种写法：xml&lt;mapper namespace="demo.dao.DepartmentMapper"> &lt;resultMap id="BaseResultMap" type="demo.bean.po.Department"> &lt;id ... /> &lt;result ... /> ... &lt;collection column="deptno" javaType="java.util.List" ofType="demo.bean.po.Employee" property="employeeSet" select="demo.dao.EmployeeMapper.selectByPK"/> &lt;/resultMap> &lt;select ...>## 3. 多对多映射MyBatis 并没有提供专门的元素用于多对多方案，采取了一种“聪明”的办法，将“三张表”的多对多关系转化为“两张表”的一对多关系。所以，在多对多关系中，使用的仍然是 collection 元素。例如：xml&lt;mapper namespace="demo.dao.DepartmentMapper"> &lt;resultMap id="BaseResultMap" type="demo.bean.po.Department"> &lt;id ... /> &lt;result ... /> ... &lt;collection column="deptno" javaType="java.util.Set" ofType="demo.bean.po.Employee" property="employeeSet" select="demo.dao.EmployeeMapper.selectByDeptno"/> &lt;/resultMap> &lt;select ...>## 4. &lt;resultMap&gt; 的继承xml<!-- 逻辑上『父类』 -->&lt;resultMap id="BaseResultMap"
